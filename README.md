# papernote

每周跟踪智能体安全、移动安全与应用安全方向的新论文，保存结构化摘要、检索记录和去重索引。

## 内容

- `weekly/`：按周发布的中文论文动态汇总。
- `papers/index.jsonl`：论文主索引，用 DOI、arXiv ID、正式出版链接和规范化标题去重。
- `inquiries/`：与论文有关的问询记录及后续追踪。
- `industry/viewpoints.json`：行业文章与综述的观点摘要。
- `scripts/build_site.py`：从论文索引、周报和行业观点生成网页数据。
- `site/`：可按周、主题和行业观点浏览内容的静态网页。
- `templates/`：周报与问询模板。

## 网页

公开页面：<https://wyinleo.github.io/papernote/>

网页不依赖前端框架。更新周报、索引或行业观点后，可在本地重新生成页面数据：

```bash
python3 scripts/build_site.py
python3 -m http.server 8000 -d site
```

随后访问 <http://localhost:8000>。推送到 `main` 后，GitHub Actions 会重新生成
`site/data.js` 并自动发布 `site/` 到 GitHub Pages。

## 收录优先级

1. 安全四大：IEEE S&P、USENIX Security、ACM CCS、NDSS。
2. 软件工程 CCF-A 会议及其安全相关论文。
3. AI 领域 CCF-A 会议中的智能体安全、模型与系统安全论文。
4. arXiv 预印本：优先知名研究团队、学术或产业实验室以及有代表性作者；统一标为“预印”，表示尚未经过同行评审。

每周报告应优先质量而非数量。仓库不把引用量、作者知名度或单位名气当作论文质量的替代指标。

## 周报字段规范

每篇论文固定使用以下顺序，字段名称不自行变更：

1. `方向`：归入智能体安全、移动安全或应用安全，可补充一个细分方向。
2. `关键词`：紧接方向列出 3—5 个检索与主题关键词。
3. `单位与作者`：先列单位，再列该单位作者；多单位逐行列举，不把全部单位和作者混写在同一句中。
4. `出处与状态`：固定写为“文献位置｜状态”，文献位置只写会议或 arXiv；状态只使用“预印”或“接收”，其中 arXiv 一律视为“预印”。
5. `公开或更新时间`：记录可核验的公开或实质更新时间。
6. `原文链接`：提供会议官方原文、作者原文或 arXiv 原文，不再使用“标识符”字段。
7. `核心问题`：用一段完整描述交代研究背景、现有缺口和论文要解决的问题，避免只有一句抽象概括。
8. `方法与贡献`：概述研究设计、关键机制和主要贡献。
9. `方法示例`：结合论文中的案例或按论文方法构造一个简明例子，说明问题如何出现、方法如何处理；若例子由整理者归纳，应避免写成论文原有实验事实。
10. `实验与证据`：记录数据集、对象、指标和主要结果。
11. `局限与风险`：说明证据边界和外推风险。
12. `实践关系`：说明对工程、产品或防御工作的意义。
13. `推荐理由`：解释为何值得阅读。

正文以中文为主。论文标题、作者姓名、会议名称、产品名称以及确有必要的通行缩写可保留英文；其余术语优先采用准确、自然的中文表达，不增设额外的边界说明字段。

## 去重规则

按以下顺序匹配：DOI → arXiv ID → 正式论文 URL → 规范化标题。预印本后续正式发表时，更新原条目并链接两个版本，不新增重复记录。

## 文件命名

- 周报：`weekly/YYYY/YYYY-Www.md`
- 问询：`inquiries/YYYY/YYYY-MM-DD-slug.md`
