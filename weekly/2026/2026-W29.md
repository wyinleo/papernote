## 精选论文

### 1. SoK: Attack and Defense Landscape of Agentic AI Systems

- **方向**：智能体安全 / SoK
- **作者与单位**：Juhee Kim（UC Berkeley、首尔大学），Wenbo Guo（UC Santa Barbara），Dawn Song（UC Berkeley）
- **出处与状态**：USENIX Security ’26，已接收；会前版本可用
- **标识符**：暂无公开 DOI；[官方页面](https://www.usenix.org/conference/usenixsecurity26/presentation/kim-juhee-agentic)
- **核心问题**：如何统一描述 Agentic AI 的设计空间、攻击面与防御机制。
- **方法与贡献**：论文给出系统化知识框架，并总结智能体组合 LLM 与非 AI 工具后产生的独特风险及开放问题。
- **实验与证据**：SoK 的主要价值是分类和综合，而非提出单一攻击或防御并做统一基准实验。
- **局限与风险**：智能体技术演化快，分类体系可能很快需要扩展；“首个全面框架”是作者主张，应结合其纳入标准和遗漏范围评估。
- **推荐理由**：适合作为后续周报的概念底座，用来判断新论文究竟覆盖模型、记忆、工具、权限还是多智能体传播层。

### 2. MASLeak: Investigating and Exposing Intellectual Property Leakage Vulnerabilities in Multi-Agent Systems

- **方向**：多智能体安全 / 知识产权泄漏
- **作者与单位**：Liwen Wang 等（香港科技大学、中国人民大学、岭南大学）
- **出处与状态**：USENIX Security ’26，已接收；会前版本可用
- **标识符**：暂无公开 DOI；[官方页面](https://www.usenix.org/conference/usenixsecurity26/presentation/wang-liwen)
- **核心问题**：仅通过公开 API，攻击者能否黑盒恢复多智能体系统的拓扑、系统提示、任务指令和工具用法。
- **方法与贡献**：MASLEAK 借鉴蠕虫传播思路构造恶意查询，使泄漏内容在智能体间被诱导、传播并保留；作者还构建了 810 个合成多智能体应用的数据集，并测试 Coze、CrewAI 等真实系统。
- **实验与证据**：作者报告多数场景下系统提示与任务指令提取成功率平均 87%，系统架构提取成功率 92%。
- **局限与风险**：合成应用与真实生产编排的差距需要进一步审查；成功率依赖攻击目标、输出可观测性与后端模型，不能直接外推到所有 MAS。
- **推荐理由**：把“提示泄漏”提升到系统架构与协作协议层，直接关联 Agent 平台的商业机密保护和最小披露设计。

### 3. Network-Level Prompt and Trait Leakage in Local Research Agents

- **方向**：智能体隐私 / 网络侧信道
- **作者与单位**：Hyejun Jeong、Mohammadreza Teymoorianfard、Abhinav Kumar、Amir Houmansadr、Eugene Bagdasarian（马萨诸塞大学阿默斯特分校）
- **出处与状态**：USENIX Security ’26，已接收；会前版本可用
- **标识符**：暂无公开 DOI；[官方页面](https://www.usenix.org/conference/usenixsecurity26/presentation/jeong)
- **核心问题**：被动网络观察者能否仅凭访问 IP/域名及时间模式，推断本地 Web/Research Agent 的用户问题和长期用户特征。
- **方法与贡献**：论文构建真实与合成人物查询的流量数据集，并提出 OBELS 指标衡量原始提示与推断提示在功能和领域知识上的相似度。
- **实验与证据**：研究智能体每个请求访问约 70—140 个域；作者报告攻击可恢复超过 73% 的提示功能/领域知识，多会话下最多恢复 32 个潜在特征中的 19 个；限制域多样性或混淆流量使攻击效果平均降低 29%，且作者称效用影响很小。
- **局限与风险**：指标与“恢复百分比”不等同于逐字重建；效果对代理策略、缓存、加密 DNS、网络路径和用户行为分布敏感。
- **推荐理由**：对“本地运行即隐私安全”的直觉形成重要反例，也直接影响自动研究、搜索与企业情报智能体的部署架构。

### 4. Bridges to Self: Silent Web-to-App Tracking on Mobile via Localhost

- **方向**：移动安全 / Web—App 隔离 / 隐私
- **作者与单位**：Tim Vlummens 等（KU Leuven、IMDEA Networks、Radboud University）
- **出处与状态**：USENIX Security ’26，已接收；会前版本可用
- **标识符**：暂无公开 DOI；[官方页面](https://www.usenix.org/conference/usenixsecurity26/presentation/vlummens)
- **核心问题**：网站能否利用 localhost 通道与原生 App 通信，把 Web Cookie 与长期原生标识符绑定。
- **方法与贡献**：结合美国和欧盟视角的大规模网页抓取与 Android App 系统分析，刻画 HTTP(S)、WebSocket、WebRTC 等标准被用于跨上下文跟踪的机制，并分析浏览器修复与 Local Network Access 权限方案。
- **实验与证据**：作者报告 Meta 与 Yandex 曾使用这类桥接；该技术可绕过清 Cookie、无痕模式、广告 ID 重置、VPN 及工作/个人配置隔离，并指出 WebRTC 全局单播 IPv6 和 `.local` mDNS 仍可能形成旁路。
- **局限与风险**：生态修复持续变化，具体可利用性需结合当前浏览器和 App 版本复测；法律分析与技术结论应分开理解。
- **推荐理由**：真实世界影响强，揭示“浏览器沙箱”和“App 沙箱”分别成立时，跨生态组合仍可能破坏隐私边界。

### 5. Cracks in the Walled Garden: Dissecting the Gray-Market of Unauthorized iOS App Distribution via Ad Hoc Sideloading

- **方向**：iOS 安全 / 应用分发 / 供应链
- **作者与单位**：Yijing Liu、Yiming Zhang、Baojun Liu、Haixin Duan（清华大学、BNRist）
- **出处与状态**：USENIX Security ’26，已接收；会前版本可用
- **标识符**：暂无公开 DOI；[官方页面](https://www.usenix.org/conference/usenixsecurity26/presentation/liu-yijing)
- **核心问题**：Ad Hoc provisioning 如何演化为绕过 App Store 审核的灰色分发产业链。
- **方法与贡献**：以用户视角收集并分析签名站点、签名工具和 `.ipa` 分发样本，反向工程证书流通与应用修改手法。
- **实验与证据**：作者识别 3,359 个活跃签名站点，逆向 12 个签名工具，获得 8,216 个 `.ipa` 条目，并报告证书转售利润率最高达 3,000%；大量样本是注入动态库的正版 App 修改版。
- **局限与风险**：研究重点是中国市场，地域和时间外推需谨慎；样本发现渠道可能遗漏封闭社群，风险类别也不代表每个样本都已被证实恶意。
- **推荐理由**：将单个恶意 App 问题扩展到证书、工具、分发站与修改包构成的供应链，适合移动应用风控和平台治理研究。

### 6. The State of Passkeys: Studying the Adoption and Security of Passkeys on the Web

- **方向**：应用安全 / 身份认证 / WebAuthn
- **作者与单位**：Louis Jannett 等（Ruhr University Bochum、Heilbronn University of Applied Sciences、University of Wuppertal）
- **出处与状态**：USENIX Security ’26，已接收；会前版本可用
- **标识符**：暂无公开 DOI；[官方页面](https://www.usenix.org/conference/usenixsecurity26/presentation/jannett)
- **核心问题**：Passkey 在真实网站的采用情况及其 WebAuthn 实现是否安全。
- **方法与贡献**：PASSKEYS-RADAR 汇聚社区目录、Tranco、CrUX 和历史网页数据；PASSKEYS-ATTACKER 可操纵 WebAuthn 各阶段消息并实现 15 类攻击测试。
- **实验与证据**：作者分析 872 个启用 Passkey 的网站，并在 103 个网站上评估攻击；18 个网站存在两类关键严重度风险，53 个至少存在一类高严重度风险，影响包括账户接管、删除 Passkey、锁定和会话固定。
- **局限与风险**：扫描是时间截面，网站可能已修复；CVSS 严重度和实际可利用条件不完全等价，测试站点也不是全体部署的随机样本。
- **推荐理由**：说明“采用抗钓鱼标准”并不自动等于安全，实现、生命周期管理和恢复流程同样关键。

### 7. Patch-Guided Vulnerability Detection: Extracting Java API Security Rules via Attack–Defense Cross-Analysis

- **方向**：应用安全 / 程序分析 / Java API 误用
- **作者与单位**：Bofei Chen、Shuang Liao、Lei Zhang、Yuan Zhang（复旦大学），Chibin Zhang、Mathias Payer（EPFL）
- **出处与状态**：USENIX Security ’26，已接收；会前版本可用
- **标识符**：暂无公开 DOI；[官方页面](https://www.usenix.org/conference/usenixsecurity26/presentation/chen-bofei)
- **核心问题**：能否从已确认安全补丁中抽取精确 API 安全规则，再发现其他应用中的同类误用。
- **方法与贡献**：VulGenie 使用补丁图隔离约束与防御修改，通过攻防交叉验证合成规则，并以偏差引导静态分析扩展检测。
- **实验与证据**：在 150 个近期 Java 安全补丁上抽取 198 条规则，作者报告精度 81.82%，其中 177 条未被 CodeQL 覆盖；在 10 个流行 Java 应用中发现 46 个零日漏洞，25 个已修复并获分配 10 个 CVE。
- **局限与风险**：评估规模和项目选择影响外部有效性；“CodeQL 未覆盖”不代表 CodeQL 无法通过新规则覆盖，零日确认程度需结合披露与补丁核验。
- **推荐理由**：把补丁知识自动转化为可执行检测规则，连接软件工程中的补丁挖掘与应用安全中的可扩展漏洞发现。

### 8. Opossum Attack: Application Layer Desynchronization using Opportunistic TLS

- **方向**：应用层协议安全 / TLS / Web 安全
- **作者与单位**：Robert Merget 等（Technology Innovation Institute、Ruhr University Bochum、Paderborn University）
- **出处与状态**：USENIX Security ’26，已接收；会前版本可用
- **标识符**：暂无公开 DOI；[官方页面](https://www.usenix.org/conference/usenixsecurity26/presentation/merget)
- **核心问题**：同时支持隐式 TLS 与机会式 TLS 时，协议升级语义能否导致请求—响应错位并破坏认证。
- **方法与贡献**：论文提出不依赖具体实现 Bug 的应用层反同步攻击，并详细分析 HTTP 的四类利用方向。
- **实验与证据**：IPv4 范围扫描发现超过 300 万服务器同时支持相关模式；HTTP 场景中发现 20,121 台服务器在 35 个端口支持机会式 HTTP，其中 2,268 台也支持 HTTPS，539 台使用相同域名，构成作者定义的可利用场景。
- **局限与风险**：暴露面数字不等于已确认成功利用；真实攻击仍依赖网络中间人位置、客户端/服务器行为及部署配置。
- **推荐理由**：提醒工程团队：即使 TLS 算法和实现本身现代且正确，协议组合与状态机语义仍会破坏认证保证。
