### 1. Overcoming the Retrieval Barrier: Indirect Prompt Injection in the Wild for LLM Systems

- **方向**：智能体安全 / 检索增强生成安全
- **关键词**：间接提示注入、检索屏障、向量嵌入、黑盒优化、工具调用
- **单位与作者**：
  - 穆罕默德·本·扎耶德人工智能大学：Hongyan Chang、Ergute Bao、Xinjian Luo、Ting Yu
- **出处与状态**：USENIX Security 2026｜接收
- **公开或更新时间**：截至 2026-07-21，官方会前原文可访问
- **原文链接**：[会议官方页面](https://www.usenix.org/conference/usenixsecurity26/presentation/chang-hongyan)；[官方会前原文](https://www.usenix.org/system/files/conference/usenixsecurity26/sec26_prepub_chang.pdf)
- **核心问题**：间接提示注入通常假设恶意文档已经进入大语言模型的上下文，但现实系统会先用检索器筛选文档。攻击文本即使能够操纵模型，如果与用户查询的语义相似度不够，也可能永远不会被取回。论文要解决的是：攻击者只能写入一个语料条目、只能调用黑盒向量嵌入接口时，如何让承载恶意指令的文档稳定越过检索屏障，并最终影响智能体行为。
- **方法与贡献**：作者把注入文档拆成两部分：一部分承载攻击目标，另一部分是提高查询相似度的短触发片段。随后用交叉熵方法在离散词元空间中反复采样、评分和更新触发片段，从而在不知道嵌入模型内部参数的情况下优化检索排名。贡献在于把攻击从“模型已读到恶意文本”的局部假设，扩展为查询、检索、阅读和执行组成的端到端链路。
- **方法示例**：假设攻击者希望用户询问“如何配置 SSH 密钥”时，系统检索到一封带有外泄指令的恶意邮件。单纯插入恶意指令可能与该问题不够相似；论文方法会保留邮件中的攻击内容，同时自动搜索一段与目标查询高度相似的短前缀，将两者拼接后写入语料库，使该邮件更可能进入检索结果，再由后续提示注入诱导智能体执行命令。
- **实验与证据**：论文覆盖 11 个信息检索基准和 8 个开源或闭源嵌入模型，报告的检索成功率接近 100%；针对 OpenAI 嵌入接口构造一个目标查询的最低成本约为 0.21 美元。在合成邮件和多智能体工作流中，一封毒化邮件使 GPT-4o 执行 SSH 密钥外泄脚本的成功率超过 80%。
- **局限与风险**：实验主要针对基于向量嵌入的检索，没有系统覆盖混合检索和重排器；不同架构嵌入模型之间的迁移效果不稳定；端到端实验使用公开基准和合成语料，不能等同于已经攻破真实生产系统。
- **实践关系**：检索层不能只依赖相似度和前若干项结果。工程上应隔离外部写入源的权限与来源，并在检索后继续保留内容信任标记、工具调用授权和敏感数据出站控制。
- **推荐理由**：它把间接提示注入研究推进到完整的“查询—检索—执行”链路，论文状态明确、实验覆盖较广，是本期最值得复现的一篇。

### 2. Insecure Coding Preferences in Long-Term Memory: Security Risks for LLM-based Code Generation

- **方向**：应用安全 / 安全代码生成
- **关键词**：长期记忆、不安全偏好、代码生成、漏洞评测、安全过滤
- **单位与作者**：
  - 南京大学：Yuchen Chen、Yuan Xiao、Chunrong Fang、Baowen Xu、Zhenyu Chen
  - 南京航空航天大学：Wei Cheng
  - 阿尔伯塔大学及阿尔伯塔机器智能研究院：Zhou Yang
  - 新加坡管理大学：Weifeng Sun、David Lo
  - 南通大学：Xiang Chen
- **出处与状态**：ISSTA 2026｜接收
- **公开或更新时间**：arXiv 版本于 2026-07-20 公开
- **原文链接**：[arXiv 原文](https://arxiv.org/abs/2607.17619)
- **核心问题**：代码助手会把用户偏好写入长期记忆，并在之后的独立会话中重复使用。某些偏好起初可能只是为了调试方便、兼容旧系统或快速通过测试，但其中可能隐含弱随机数、硬编码凭据或不安全解析等模式。论文研究这些不安全偏好是否会跨任务持续影响代码生成，以及模型给出安全警告、遵循显式安全要求和保持功能正确性之间存在怎样的权衡。
- **方法与贡献**：作者在 ChatGPT、Gemini、Qwen 和 Grok 上，针对 SALLM 的 100 个 Python 任务以及 CWEval 的 94 个其他语言任务，构造与常见弱点枚举相对应的不安全偏好记忆。研究同时测量功能通过率、漏洞率、安全警告率和记忆引用率，并比较显式安全提示及写入前过滤器的缓解效果。
- **方法示例**：论文给出的偏好模式之一，是要求以后用 `random.randint(1, 100)` 生成会话标识。该写法可能让当前示例更容易运行，却会把可预测、取值空间很小的随机数模式保存为长期偏好。后续遇到新的会话管理任务时，模型可能继续套用这一模式。研究据此比较无记忆、写入风险记忆、加入安全要求和写入前过滤等条件下生成代码的安全性。
- **实验与证据**：作者报告，在 5 种编程语言上，加入不安全偏好记忆后漏洞率上升 2.7—50.3 个百分点，而不少场景中的功能正确性反而提高。显式安全要求可使漏洞率下降 19.7—33.6 个百分点，但最多损失 15.9 个百分点的功能通过率。两个大语言模型过滤器在 57 条已标注风险记忆上均检出了全部风险项。
- **局限与风险**：闭源服务的模型和策略可能随时间变化；每个输入只生成 3 个样本；风险记忆由研究者从已知不安全解法中提炼，可能比自然用户偏好更集中；过滤器结果来自规模较小、分布相近的标注集合，不能外推为通用保证。
- **实践关系**：代码助手的记忆写入应进入安全评审和审计轨迹，不能把用户偏好自动提升为跨项目的高信任规则。与其只在生成后扫描代码，更适合在长期记忆写入前识别、提示和阻断风险模式。
- **推荐理由**：论文同时考察安全和功能效用，揭示了“代码更容易通过测试，但安全性反而下降”的危险错觉，对集成开发环境和编程智能体的记忆设计有直接参考价值。

### 3. Agent Data Injection Attacks are Realistic Threats to AI Agents

- **方向**：智能体安全 / 可信数据边界
- **关键词**：数据注入、网页智能体、编程智能体、结构伪造、信息流
- **单位与作者**：
  - 首尔大学：Woohyuk Choi、Juhee Kim、Taehyun Kang、Byoungyoung Lee
  - Largosoft：Jihyeon Jeong
  - 伊利诺伊大学厄巴纳—香槟分校：Luyi Xing
- **出处与状态**：arXiv｜预印
- **公开或更新时间**：2026-07-06 公开；本期补录
- **原文链接**：[arXiv 原文](https://arxiv.org/abs/2607.05120)
- **核心问题**：现有防御通常试图区分“可信指令”和“不可信文本”，但智能体上下文中还包含作者身份、网页元素编号、工具调用记录等结构化数据。即使模型不把网页正文当作指令，它仍可能把攻击者控制的数据误认成可信元数据。论文要说明这种数据边界混淆是否形成独立于传统提示注入的攻击面，并评估现有信息流防御为什么仍可能失效。
- **方法与贡献**：论文提出智能体数据注入攻击，并利用概率分隔符注入，让经过转义或格式化后的攻击文本仍可能被模型解释为结构边界。作者在网页智能体和编程智能体中构造任意点击、伪造来源诱导命令执行、伪造工具结果触发供应链风险等场景，并扩展 AgentDojo 基准以比较传统提示注入与数据注入。
- **方法示例**：网页智能体通常会把页面内容整理成带元素编号的文本，再要求模型选择要点击的编号。攻击页面可以在普通内容中嵌入近似系统分隔符和伪造编号，让模型误以为攻击者写入的按钮是可信解析器生成的元素。论文方法通过多种分隔符变体反复测试这种结构混淆，并观察智能体是否点击攻击者指定对象。
- **实验与证据**：在 6 个模型、157 个 JSON 和网页文档对象模型测试用例上，作者报告 JSON 场景攻击成功率为 31.3%—43.3%，网页场景为 33.3%—100%。扩展后的 AgentDojo 包含 96 个用户任务、108 个数据注入攻击和 935 个传统提示注入；所测防御把传统注入压到 0—0.7%，但数据注入仍最高达到 50%。只有严格信息流方案把数据注入降为 0%，同时正常任务效用降至 36.5%。
- **局限与风险**：攻击通常假设对手知道或能恢复智能体使用的数据格式；闭源服务的内部格式未必能稳定提取；智能体层评估只使用 GPT-5.2；真实产品中的确认对话框、版本变化和厂商修复会影响攻击效果。
- **实践关系**：工具返回值应尽量采用带类型对象或不透明句柄，而不是直接拼接文本。可信元数据与用户内容需要保持不可伪造的来源标记，敏感工具调用还应由独立策略执行器再次验证。
- **推荐理由**：论文指出“指令与数据分离”仍不够细，智能体数据内部同样需要字段级的信任和完整性边界，适合安全架构和工具协议设计者阅读。

### 4. Adaptive Adversaries: A Multi-Turn, Multi-LLM Benchmark for LLM Agent Security

- **方向**：智能体安全 / 自适应红队评测
- **关键词**：多轮攻击、多模型评测、自适应对手、攻击成功率、结果判定
- **单位与作者**：
  - Lambda：Devina Jain、David Hartmann、Chuan Li
- **出处与状态**：arXiv｜预印
- **公开或更新时间**：2026-07-20 公开
- **原文链接**：[arXiv 原文](https://arxiv.org/abs/2607.18063)
- **核心问题**：许多安全基准使用固定攻击提示，并只统计一次交互中的成功率，但现实攻击者会观察拒绝理由、调整措辞和更换策略。固定攻击池可能低估模型经过多轮试探后暴露的弱点，单一攻击模型和总攻击成功率也可能掩盖场景差异。论文因此研究多轮、自适应、多攻击模型评测会如何改变对智能体安全性的判断。
- **方法与贡献**：作者设计了 21 个攻击场景组成的攻击者—防御者协议。攻击模型最多调整 15 轮，每一轮的防御回答则作为新的无记忆交互进行评估，并用结构化结果检查器判断攻击是否真正达成目标。项目还提供场景、编排器、命令行工具、交互记录和重放数据。
- **方法示例**：在固定评测中，一条越权请求被拒绝后就记为防御成功；在该论文的协议中，攻击模型会读取拒绝回答，下一轮可能把同一目标拆成看似无害的子任务、改变角色设定或利用前一轮暴露的策略边界。系统持续记录每轮尝试，并由结果检查器判断是否真的发生越权行为，而不是只根据回答语气判断。
- **实验与证据**：3 个攻击模型与 3 个防御模型组成 945 场对抗。只计算第一轮时，攻击成功率为 0—1%；允许 15 轮自适应后上升至 5.4%—14%。合并 3 个攻击模型发现的独特成功攻击，比最佳单一攻击模型多 1.4—2.2 倍。21 个场景中有 13 个能区分至少一对防御模型，但场景间排名一致性较低，Kendall 协调系数为 0.19。
- **局限与风险**：每个组合只有 5 次重复，置信区间较宽；21 个场景经过难度校准，并非生产任务的随机样本；防御模型不保留跨轮记忆，与真实有状态智能体不同；闭源接口变化也会降低逐词元复现能力。
- **实践关系**：智能体安全回归测试应加入实时自适应攻击者、多个攻击模型、场景级结果和基于实际结果的判定器，不能只依赖静态越狱集合或汇总攻击成功率。
- **推荐理由**：论文提供了可执行的红队评测设计，适合用来改造内部测试流水线；但样本规模决定了它更适合作为评测框架，而不是模型安全排行榜。

### 5. Bad Memory: Evaluating Prompt Injection Risks from Memory in Agentic Systems

- **方向**：智能体安全 / 持久记忆安全
- **关键词**：跨会话攻击、记忆污染、仓库指令、攻击载荷持久性、编程智能体
- **单位与作者**：
  - 华盛顿大学：Soham Gadgil、David Alexander、Sai Sunku、Franziska Roesner
- **出处与状态**：arXiv｜预印
- **公开或更新时间**：2026-07-16 公开；本期补录
- **原文链接**：[arXiv 原文](https://arxiv.org/abs/2607.14611)
- **核心问题**：Claude Code、Codex 等编程智能体会自动读取 `CLAUDE.md`、`AGENTS.md` 或其他长期行为与知识文件。一旦恶意指令进入这些文件，风险可能不只发生在当前会话，还会持续影响未来会话、较弱模型或子智能体。论文希望同时回答两个问题：预植入记忆中的攻击载荷能否持续奏效，以及模型本轮拒绝攻击是否意味着风险已经被清除。
- **方法与贡献**：作者在沙箱合成工作区中设置凭据外泄、未授权工具使用和品牌定向三类目标，并比较单次探测、重复同一攻击、叠加不同攻击三种会话序列。评估除攻击成功率外，还单独测量攻击载荷是否继续保留在记忆中，从而区分“当前拒绝”和“未来不再受影响”。
- **方法示例**：攻击者预先在仓库的 `AGENTS.md` 中加入一条要求把环境凭据发送到外部地址的伪装说明。第一个会话中的模型可能识别并拒绝执行，但若该说明仍留在文件中，下一次启动的较弱模型会重新读取它并再次面临攻击。论文通过连续会话重复或叠加此类载荷，分别记录执行结果和文件中的载荷是否仍存在。
- **实验与证据**：研究测试 4 个模型，每种条件重复 10 次。单次探测的平均攻击成功率为 23.3%—63.3%。叠加不同攻击时，Opus 4.7 的平均攻击成功率最低，为 18.3%，但攻击载荷保留率最高，达到 93.3%，说明本轮拒绝与清除未来风险是两个不同指标。
- **局限与风险**：工作区和任务均为合成；主要实验假设攻击载荷已经存在于记忆中，初步实验没有可靠证明外部内容会被自动写入可信记忆；研究只覆盖 3 类目标、较短会话序列和每条件 10 次重复，结果也会受到具体产品版本和策略变化影响。
- **实践关系**：智能体启动时应校验高影响记忆文件；记忆写入应展示来源、差异和用户确认。模型拒绝攻击后还应触发隔离或修复，而不是让载荷继续留给后续会话、弱模型或子智能体。
- **推荐理由**：论文把安全指标从单轮攻击成功率扩展到跨会话持久性，特别适合评估会自动加载仓库指令文件的编程智能体。

### 6. Self-State Attacks on Self-Hosted AI Agents: How Far Can OS Defenses Go?

- **方向**：智能体安全 / 主机与状态完整性
- **关键词**：自托管智能体、自状态攻击、操作系统防御、文件完整性、备份恢复
- **单位与作者**：
  - 阿卜杜拉国王科技大学生成式人工智能卓越中心：Yimeng Chen、Nathanaël Denis、Roberto Di Pietro、Jürgen Schmidhuber
  - 瑞士人工智能实验室 IDSIA-USI/SUPSI：Jürgen Schmidhuber
- **出处与状态**：arXiv｜预印
- **公开或更新时间**：2026-07-20 公开
- **原文链接**：[arXiv 原文](https://arxiv.org/abs/2607.17986)
- **核心问题**：自托管智能体为了学习和适应，必须合法读写自己的记忆、指令与配置；攻击者污染这些文件时，往往也使用相同的文件写入机制。如果恶意更新与正常适应在操作系统层表现为相同系统调用，那么传统访问控制和文件监控能识别多少攻击，又有哪些风险必须依赖更高层语义才能判断，是论文试图划定的防御边界。
- **方法与贡献**：论文用“目标、机制、粒度、时间”四个维度组织自状态攻击空间，在一个代表性自托管智能体上构造 23 类攻击单元和 43 个真实文件操作。作者采集编程、研究、运维和通用四类工作负载轨迹，再比较访问控制、文件完整性监测、基于工作负载的异常检测以及备份恢复。
- **方法示例**：智能体为了记录用户偏好，可能合法地向记忆文件追加一句话；攻击者也可能通过同一次写入接口追加一条改变工具权限的指令。只看“哪个进程写了哪个文件”，两次操作可能非常相似。论文的方法会先按指令、配置和可变记忆区分保护目标，再用正常工作负载建立写入行为基线，对异常修改告警，并用快照保证受污染状态可以回滚。
- **实验与证据**：在每个工作负载和攻击条件重复 30 次后，23 类攻击中有 11 类在操作系统层可见、8 类仅在特定条件下可见、4 类无法区分。全量文件完整性监测的真正率为 100%，但合法写入的逐事件误报率达到 0.76—0.99；按工作负载建模后，误报率降至 0.08—0.36，仍会漏掉与正常记忆写入高度重叠的小改动。
- **局限与风险**：研究只测试一个智能体实现和四类工作负载；数据较少的行为模型仍有较高误报率；操作系统事件无法理解文件内容的语义，因此不能判断一次合法写入究竟是在学习还是在污染状态。
- **实践关系**：应把指令、配置和可变记忆分层保护：前两者尽量只读或通过受控流程更新，记忆层结合工作负载监控与可回滚快照。文件权限或完整性监测都不能单独解决语义等价写入问题。
- **推荐理由**：论文用经典主机安全机制解释智能体自修改风险，有助于把长期记忆研究转化为可部署的系统防御方案。

### 7. Fuzz’EMup: Leveraging EM Side-Channel Emanation to Guide Black-Box Embedded Firmware Fuzzing

- **方向**：应用安全 / 嵌入式固件安全
- **关键词**：黑盒模糊测试、电磁侧信道、固件覆盖反馈、执行轨迹、微控制器
- **单位与作者**：
  - 克莱姆森大学：Fatemeh Moradihaghighi、Mashrur Chowdhury、Zhenkai Zhang
  - 得克萨斯理工大学：Zihao Zhan
  - 罗切斯特大学：Yanan Guo
  - 美国东北大学：Ziming Zhao
- **出处与状态**：arXiv｜预印
- **公开或更新时间**：2026-07-18 04:25（北京时间）公开
- **原文链接**：[arXiv 原文](https://arxiv.org/abs/2607.16487)
- **核心问题**：许多嵌入式设备无法提取固件、插入覆盖探针或稳定重宿主，传统覆盖引导模糊测试因此失去最重要的反馈信号。纯随机黑盒测试又很难判断一个新输入是否触发了新的执行路径。论文研究能否从微控制器运行时产生的电磁辐射中提取近似覆盖反馈，并用它指导输入选择和种子调度。
- **方法与贡献**：作者先比较设备活动和空闲状态，选择最能反映执行差异的频段；随后用动态时间规整对齐存在抖动的电磁轨迹，检测轨迹持续分叉的位置；最后按分叉时间组织执行树，减少每条新轨迹与全部历史轨迹逐一比较的成本，并把有新颖分叉的输入优先放回模糊测试队列。
- **方法示例**：向同一固件发送两个串口输入后，示波器采集到的电磁波形前半段相似，但其中一个输入在某个时间点开始持续偏离，可能意味着程序进入了不同分支。系统先对齐两个波形以消除时序抖动，再把持续分叉记为新的执行特征，并提高该输入作为后续变异种子的优先级，而不需要读取固件内部覆盖计数器。
- **实验与证据**：论文在同一 STM32L476RG 微控制器上运行 GPS、步进电机、数控设备和焊台四个真实固件。硬件执行阶段只使用电磁信号反馈，再在自建 QEMU 板级模型中重放输入以统计真实覆盖。电磁引导方法在四个目标上覆盖更多指令，覆盖新颖性的命中率提高 1.19—2.58 倍，并以更少输入达到随机基线的最终覆盖水平。
- **局限与风险**：四个目标均移植到同一微控制器并通过串口测试，尚未验证实时操作系统、网络协议、不同封装和现场噪声；QEMU 重放与物理执行可能存在语义偏差；论文展示的是覆盖提升而非新漏洞，且没有充分报告多次独立硬件实验的置信区间。
- **实践关系**：该方法适合“拿不到固件但能重复输入”的硬件实验室评估。实际使用仍需把电磁轨迹新颖性与崩溃、超时和外部状态异常等判定信号结合，并计入采样设备和轨迹对齐带来的吞吐成本。
- **推荐理由**：把物理侧信道转化为黑盒固件测试反馈，思路新颖且与物联网设备安全直接相关；现有证据更适合支持可行性，而不能直接证明漏洞发现能力。
